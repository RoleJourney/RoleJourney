# RoleJourney - Projeto: Plataforma de Roadmaps de Carreira Baseados em Vagas

## 1. Visão Geral

Este projeto consiste no desenvolvimento de uma plataforma SaaS que utiliza Inteligência Artificial para gerar **roadmaps de aprendizado personalizados baseados em objetivos profissionais reais**, como vagas específicas ou setores de atuação.

A plataforma tem como objetivo resolver um dos maiores problemas enfrentados por estudantes e profissionais em transição de carreira:

> Não saber exatamente **o que estudar, em que ordem estudar, e como validar o conhecimento adquirido para alcançar uma vaga específica**.

Diferente de plataformas tradicionais de aprendizado que oferecem cursos isolados ou trilhas genéricas, este produto propõe um **sistema orientado a objetivos**, no qual o usuário define um destino profissional e recebe um plano estruturado para chegar até ele.

---

# 2. Problema

Estudantes e profissionais enfrentam diversas dificuldades ao tentar evoluir na carreira:

* Não sabem quais habilidades são realmente necessárias para determinada vaga.
* Não sabem qual ordem seguir no aprendizado.
* Não conseguem avaliar se já estão prontos para aplicar para vagas.
* Consomem cursos e conteúdos de forma desorganizada.
* Não possuem feedback prático sobre seus projetos.

Além disso, o mercado muda constantemente, o que torna muitos roadmaps tradicionais rapidamente desatualizados.

---

# 3. Proposta de Solução

A plataforma oferece um sistema de **planejamento de carreira orientado por IA**, que transforma objetivos profissionais em um caminho estruturado de aprendizado.

Fluxo principal:

1. O usuário cria uma conta.
2. Define um objetivo profissional (exemplo: uma vaga específica ou área de atuação).
3. Envia seu currículo ou perfil profissional.
4. A IA analisa o objetivo e o perfil do usuário.
5. O sistema identifica lacunas de habilidades (skill gaps).
6. Um roadmap personalizado é gerado.
7. O usuário segue o roadmap através de etapas, projetos e desafios.

---

# 4. Diferencial Central

O grande diferencial do produto é que ele funciona como um **GPS de carreira**, onde:

* O destino é definido pelo usuário.
* O sistema calcula o melhor caminho até esse destino.
* O caminho pode ser atualizado conforme o progresso do usuário.

---

# 5. Principais Funcionalidades

## 5.1 Definição de Objetivo Profissional

O usuário pode definir seu objetivo de diversas formas:

* Colando um link de vaga
* Descrevendo um cargo desejado
* Selecionando um setor ou área profissional

Exemplos:

* Backend Developer
* Data Scientist
* UX Designer
* DevOps Engineer

---

# 5.2 Análise de Currículo

O usuário pode enviar:

* currículo em PDF
* texto
* lista de habilidades

A IA analisa:

* habilidades atuais
* experiências
* tecnologias conhecidas
* nível aproximado de conhecimento

---

# 5.3 Análise de Skill Gap

O sistema compara:

Perfil atual do usuário
versus
Requisitos da vaga ou carreira desejada

Resultado:

Lista de habilidades faltantes.

Exemplo:

Usuário possui:

* HTML
* CSS
* JavaScript

Vaga exige:

* Node.js
* Docker
* Testing
* APIs REST

O sistema identifica o gap.

---

# 5.4 Geração de Roadmap

Com base nas lacunas de habilidades, a IA gera um roadmap estruturado contendo:

* etapas de aprendizado
* ordem recomendada
* dependências entre habilidades

Exemplo de estrutura:

Etapa 1 – Fundamentos
Etapa 2 – Ferramentas principais
Etapa 3 – Projetos práticos
Etapa 4 – Preparação para entrevistas

---

# 5.5 Roadmap Visual Interativo

O roadmap é exibido como um **mapa visual interativo**, com nós conectados representando:

* habilidades
* conceitos
* projetos
* etapas

Isso permite que o usuário visualize o progresso de forma clara e compreenda a relação entre os conhecimentos.

---

# 5.6 Projetos Práticos

Para cada etapa do roadmap, a plataforma sugere projetos práticos.

Cada projeto inclui:

* descrição
* requisitos
* objetivos
* checklist de funcionalidades

Exemplo:

Projeto: API de autenticação

Objetivos:

* implementar login
* implementar registro
* criar sistema de autenticação
* armazenar usuários em banco de dados

---

# 5.7 Sistema de Progresso

O usuário pode marcar:

* habilidades aprendidas
* projetos concluídos
* etapas finalizadas

Isso atualiza automaticamente o progresso geral.

Exemplo:

Progresso total: 45%

---

# 5.8 Feedback Automatizado

A plataforma pode analisar projetos do usuário e oferecer feedback automático sobre:

* qualidade de código
* estrutura do projeto
* organização
* boas práticas

Isso ajuda o usuário a melhorar continuamente.

---

# 6. Roadmap Iterativo

Uma característica importante da plataforma é que os roadmaps **não são estáticos**.

Eles podem se adaptar com base em:

* progresso do usuário
* novas habilidades adquiridas
* mudanças no objetivo profissional

Esse conceito cria um sistema de aprendizado **iterativo e adaptativo**.
